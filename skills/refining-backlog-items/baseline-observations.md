# Baseline Observations (RED)

In the absence of a dedicated backlog-refinement skill, common failure modes include:

- Treating the task as “copyediting” instead of making items deliverable-anchored (artifact + acceptance).
- Treating rough prose as complete; not enriching with expert perspective or surfacing gaps/pitfalls.
- Rewriting descriptions without tightening scope to a single objective.
- Leaving Parent Issue titles inconsistent, overly long, or not outcome-oriented.
- Keeping Definitions of Done subjective (“validated”, “reviewed”, “complete”) or non-verifiable.
- Skipping clarifying questions entirely, even when record selection is ambiguous or when acceptance criteria depend on missing constraints.
- Asking clarifying questions but still writing any `./scratchpaper/refined-backlog-...md` draft before waiting for answers.
- Returning a refined list in chat only, without producing the requested markdown artifact (default `./scratchpaper/...md`) or without updating a referenced table in place.
- Producing markdown tables that are hard to copy/paste when DoD contains multi-bullet lists.
- Over-explaining internal terms of art/proprietary context (tickets read like documentation).

These are the target failures the skill must prevent.
